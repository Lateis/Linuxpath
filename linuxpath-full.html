Lesson 2 — Linux
What is Linux?
The operating system that runs the world
Linux is the backbone of modern technology. It's free, open, and incredibly powerful. Let me explain why cybersecurity pros absolutely love it!
The Basics

An operating system is software that manages your computer's hardware and lets you run programs. You're probably familiar with Windows or macOS — Linux is another operating system, but it's open source, meaning anyone can see and improve its code.

🌐

Powers the Internet

Over 96% of web servers run Linux. Every time you visit a website, Linux is likely serving it.

🔓

Open Source

The code is public. Security researchers can inspect it, find bugs, and make it safer.

🛡️

Cybersecurity Standard

Kali Linux, Parrot OS, and Ubuntu are all Linux — the preferred platforms for security testing.

⚙️

Full Control

You control every part of the system. No hidden processes, no locked features.

Why Security Professionals Choose Linux

Linux gives professionals fine-grained control over networks, processes, and files. The command line allows automation that would take hours to do manually through a GUI. Most security tools are built for Linux first — and some exclusively run on Linux.

Lesson 4 — Command Line
What is the Command Line?
Text-based control — the professional's interface
Forget clicking icons — the command line is where real power lives. Once you get comfortable here, you'll wonder how you ever did anything without it!
GUI vs CLI

A GUI (Graphical User Interface) is what you're used to — windows, icons, menus, and buttons. A CLI (Command Line Interface) is a text-only environment where you type instructions directly to the computer.

Both accomplish the same goals. But the CLI is dramatically faster, more precise, and capable of automating complex tasks that would take hours to do manually through a GUI.

⚡

Speed

Rename 10,000 files in one command. A GUI would take hours.

🎯

Precision

Every action is explicit. You know exactly what you're telling the computer.

🤖

Automation

Chain commands into scripts. Run complex workflows with one keystroke.

🌍

Remote Access

Manage servers on the other side of the world via SSH — all through text.

The Prompt

When you open Termux, you'll see something like ~ $ — this is the prompt. It's the terminal waiting for your instruction. Type a command, press Enter, and the system responds. That's the entire interaction model — and it's beautiful in its simplicity.

Lesson 5 — Core Commands
Essential Termux Commands
Your first toolkit — learn these and you'll feel at home
These 10 commands are your foundation. Master them and you'll be navigating Linux like a pro. Study each card carefully — there's a lot of gold here!
pwd
What it doesPrints your current location in the file system — your "working directory."
Technical DetailStands for Print Working Directory. Returns the full absolute path of the directory you're currently inside.
Example
$ pwd
Output/data/data/com.termux/files/home
Why it mattersBefore acting, you need to know where you are. Always run pwd when you feel lost.
💡 Think of pwd as GPS for your terminal. Run it often when navigating.
ls
What it doesLists all files and folders in your current directory.
Technical DetailStands for List. Use ls -la for detailed view including hidden files.
Example
$ ls
OutputDocuments Downloads projects script.py
Why it mattersYou need to see what's in a directory before working inside it.
💡 Try ls -la for file sizes, permissions, and hidden files.
cd
What it doesChanges your current directory — moves you into a different folder.
Technical DetailStands for Change Directory. Works with relative and absolute paths.
Example
$ cd Documents
OutputNo output — your prompt changes to reflect the new directory.
Why it mattersNavigation is fundamental. Everything depends on being in the right location.
💡 Type cd ~ to jump back to your home directory from anywhere.
cd ..
What it doesMoves you one level up — back to the parent directory.
Technical DetailThe .. always points to the parent. Chain: cd ../.. goes up two levels.
Example
$ cd ..
OutputNo output — you're now one folder level higher.
Why it mattersYou'll constantly move between directories. This is your "go back" button.
💡 Combine with pwd — go up, then verify where you are.
mkdir
What it doesCreates a new directory (folder) with the name you specify.
Technical DetailStands for Make Directory. Use mkdir -p to create nested directories at once.
Example
$ mkdir myproject
OutputNo output. Run ls to confirm it was created.
Why it mattersOrganisation is critical. Always structure your files in logical directories.
💡 Use underscores not spaces: mkdir my_project
rm
What it doesPermanently deletes files or directories. There is no recycle bin.
Technical DetailStands for Remove. Use rm -r to delete directories and all their contents recursively.
Example
$ rm file.txt
OutputNo output. The file is gone immediately and permanently.
Why it mattersClean up your workspace and manage disk space carefully.
⚠️ rm deletes forever. Always double-check before pressing Enter.
clear
What it doesClears the terminal screen — gives you a clean slate.
Technical DetailSends a terminal escape sequence that scrolls the viewport up. History is still accessible by scrolling.
Example
$ clear
OutputThe screen becomes clean and empty. Prompt appears at the top.
Why it mattersA cluttered terminal is hard to read. Keep it clean for clarity and focus.
💡 The shortcut Ctrl + L does the same thing faster.
pkg update
What it doesRefreshes Termux's list of available packages from online repositories.
Technical DetailDownloads updated package metadata. Does not upgrade installed packages — that's pkg upgrade.
Example
$ pkg update
OutputA list of repositories being checked with package counts.
Why it mattersKeeps your package database current so you install the latest, most secure versions.
💡 Always run pkg update before installing anything new.
pkg upgrade
What it doesUpgrades all installed packages to their latest available versions.
Technical DetailDownloads and installs newer versions of all packages where updates exist. Run after pkg update.
Example
$ pkg upgrade
OutputA list of packages being upgraded. Prompted to confirm with Y.
Why it mattersSecurity patches and bug fixes arrive via upgrades. Keeping packages updated is essential.
💡 Run pkg update && pkg upgrade to do both in one step.
pkg install
What it doesInstalls a new software package from Termux's repositories.
Technical DetailDownloads and installs the specified package plus all its dependencies automatically.
Example
$ pkg install python
OutputDownload progress, dependency resolution, and installation confirmation.
Why it mattersThis is how you extend Termux's capabilities with new tools and languages.
💡 Install multiple at once: pkg install python git curl

Lesson 8 — Ethics
Responsibility & Ethics
Knowledge is power — use it wisely
This is the most important lesson of all. The same skills that make you a great security professional can cause serious harm if misused. Let's talk about the line between legal and illegal.
The Core Rule

Cybersecurity skills are like a lockpick set — incredibly useful for locksmiths and security professionals, but dangerous in the wrong hands. You must only use these skills on systems you own or have explicit written permission to test. Anything else is a crime in virtually every country.

✓ Legal & Ethical Use

  • Testing your own devices and servers
  • Authorised penetration testing (bug bounties)
  • Learning in sandboxed lab environments
  • Studying with CTF (Capture The Flag) challenges
  • Building security tools for defence
  • Helping organisations find vulnerabilities they hire you to find

✕ Illegal & Harmful

  • Accessing any system without permission
  • Scanning networks you don't own
  • Intercepting others' communications
  • Installing tools on others' devices
  • Attempting to break into accounts
  • Sharing others' private data
The Career Path

Ethical hackers — called penetration testers — are paid very well to legally attack systems with permission, find weaknesses, and report them. This is a legitimate, growing, and highly respected career. The knowledge you're building here is the foundation of that career. Use it to build, not to destroy.
